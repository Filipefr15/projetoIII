Como entender o diagrama atualizado:

  • Frontend: O usuário acessa o painel (Dashboard) e o Chat. Requisições normais vão para a API REST, e o chat usa WebSockets para comunicação em tempo real.
  • Backend (Django): O Django é o cérebro que recebe tudo. O Django Channels mantém a conexão aberta para o chat enviar respostas de forma dinâmica.
  • Assíncrono (Celery/Redis): Como a IA demora a responder, o Django manda o trabalho pesado para os "Workers" do Celery usando o Redis exclusivamente como fila de tarefas (Broker).
  • Armazenamento: Todos os dados do sistema e contextos ficam concentrados no PostgreSQL.
  • Serviço de IA: O LangChain organiza a lógica, consulta o banco de dados (PostgreSQL) para dar contexto e pede à API do LLM para gerar a resposta final.
